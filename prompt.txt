Your Role:
Ты Senior Product Manager, Senior UX Designer, Senior Android Architect и профессиональный фотограф-пейзажист с опытом съёмки рассветов и закатов. Твоя задача — не создавать MVP, а сразу спроектировать полноценный production-ready продукт с продуманной архитектурой, UX и возможностями масштабирования.

Short basic instruction:
Разработай детальную концепцию Android-приложения для фотографов рассветов и закатов, использующего бесплатные API погодных сервисов и ИИ-рекомендации.

What you should do:
Создай полноценную концепцию приложения, включая:

- позиционирование продукта;
- целевую аудиторию;
- пользовательские сценарии;
- UX/UI-концепцию;
- структуру экранов;
- функциональные модули;
- систему персонализации;
- интеграции с API;
- архитектуру данных;
- рекомендации по реализации Android-приложения на Material Design 3;
- roadmap реализации.

Приложение должно использовать:

- OpenWeatherMap (бесплатный тариф);
- Open-Meteo;
- Яндекс Погоду (бесплатный ограниченный тариф);
- OnlySQ API для доступа к ИИ-моделям.

Your Goal:
Спроектировать красивое, удобное и максимально полезное приложение «от фотографа для фотографов», которое позволяет быстро принимать решение о съёмке рассветов и закатов и получать максимально точные рекомендации на основе погодных данных, пользовательского профиля и ИИ-анализа.

Result:
Сформируй результат в виде следующих разделов:

# 1. Концепция продукта

- название приложения;
- краткое описание;
- уникальное торговое предложение (USP);
- ключевые преимущества перед погодными приложениями общего назначения.

# 2. Целевая аудитория

Поддерживать два основных сегмента:

- фотографы рассветов;
- фотографы закатов.

Также предусмотреть возможность совмещения обоих направлений.

Для каждого пользователя в настройках должна настраиваться персональная карточка фотографа:

- уровень опыта;
- предпочитаемый стиль съёмки;
- пейзажная/городская/морская съёмка;
- используемая камера;
- используемые объективы;
- наличие штатива;
- транспортная мобильность;
- готовность ехать на локацию;
- рабочий регион;
- предпочитаемые погодные условия;
- предпочтения по композиции и освещению.

# 3. Первый запуск (Onboarding)

Опиши идеальный onboarding:

- пошаговые инструкции;
- объяснение каждой функции;
- ввод API-ключей;
- настройка профиля фотографа;
- настройка уведомлений;
- настройка источников данных;
- настройка ИИ-рекомендаций;
- возможность пропуска необязательных настроек.

Пользователь после первого запуска должен полностью подготовить приложение к работе.

# 4. Главный экран

Опиши главный экран максимально подробно.

Пользователь должен открыть приложение и сразу увидеть:

- оценку сегодняшнего рассвета;
- оценку сегодняшнего заката;
- вероятность красивого неба;
- вероятность драматичных облаков;

- рекомендации по времени выезда;
- итоговый рейтинг фотогеничности.

Опиши собственную систему оценки условий съёмки.

Придумай интегральный рейтинг от 0 до 100, учитывающий:

- облачность;
- высоту облаков;
- тип облаков;
- влажность;
- осадки;
- ветер;
- прозрачность атмосферы;
- туман;
- аэрозоли;
- температуру;
- давление;
- исторические данные;
- согласованность нескольких погодных сервисов.

Объясни формулу расчёта.

# 6. AI-рекомендации

Через OnlySQ реализовать рекомендации:

- стоит ли ехать на съёмку;
- какую локацию выбрать;
- какой объектив использовать;
- нужен ли штатив;
- предполагаемый визуальный результат;
- советы по композиции;
- советы по экспозиции;
- рекомендации по настройкам камеры.

Предусмотреть возможность полного отключения ИИ.

# 7. Персонализация

Система должна адаптировать рекомендации под:

- опыт пользователя;

- предпочтения по погоде;
